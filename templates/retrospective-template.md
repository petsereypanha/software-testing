## Action Items

*For each improvement identified, define a specific action item. Vague ideas don't get implemented.*

| # | Action Item | Owner | Due Date | Success Criteria |
|---|-------------|-------|----------|-----------------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

**Rule for action items:** Each must be:
- **Specific** — "Add QA story review to Sprint Planning agenda" not "improve planning"
- **Assigned** — one person owns it (even if others help)
- **Time-boxed** — due by a specific sprint or date
- **Measurable** — you should know when it's done
